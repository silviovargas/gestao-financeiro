Fazer backup agora

Baixa uma cópia completa e íntegra do banco de dados database.db com todos os dados do sistema — usuários, lançamentos financeiros, checklist, metas de investimento e permissões. O arquivo gerado pode ser usado para restaurar o sistema inteiro em caso de perda de dados.

O arquivo será salvo com o nome ballet_backup_AAAAMMDD_HHMM.db
🔄

Como restaurar um backup

Para restaurar, copie o arquivo .db baixado para o servidor e reinicie o serviço:

# 1. Parar o serviço
sudo systemctl stop ballet-checklist

# 2. Substituir o banco (ajuste o nome do arquivo)
cp ~/ballet_backup_20260528_0917.db /var/www/checklist-app/database.db

# 3. Reiniciar
sudo systemctl start ballet-checklist
⚠️ Restaurar um backup substitui todos os dados atuais. Faça um backup do estado atual antes de restaurar um antigo.
🖥️

Backup via terminal (automático)

Você pode agendar backups automáticos pelo terminal da VPS usando o cron:

# Backup manual a qualquer momento:
cp /var/www/checklist-app/database.db ~/ballet-backup-$(date +%Y%m%d).db

# Agendar backup diário às 02h (crontab -e):
0 2 * * * cp /var/www/checklist-app/database.db ~/backups/ballet-$(date +\%Y\%m\%d).db
📋

Recomendações

✓ Frequência: Faça backup ao menos uma vez por semana. Idealmente toda sexta-feira antes de fechar o dia.
✓ Antes de atualizações: Sempre faça backup antes de instalar patches ou atualizar o sistema.
✓ Armazenamento externo: Guarde os arquivos de backup em local diferente do servidor — pendrive, Google Drive ou Dropbox.
✓ Retenção: Mantenha os últimos 4 backups (1 mês). Backups mais antigos podem ser apagados.
